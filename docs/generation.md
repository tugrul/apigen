# Code Generation

## StubGenerator

`StubGenerator` reads a PHP interface via reflection and writes a concrete implementation class to disk.

```php
use Tugrul\ApiGen\Generator\{StubGenerator, SuffixNamingStrategy};

$generator = new StubGenerator(
    outputDir:    __DIR__ . '/src/Generated',
    naming:       new SuffixNamingStrategy('Stub'),
    pathResolver: null,   // null = auto-discover composer.json
);
```

### `generate(string $interfaceClass): GeneratedStub`

Generates a single stub. Returns a `GeneratedStub` value object.

```php
$stub = $generator->generate(UserApi::class);

echo $stub->interfaceClass;  // 'MyApp\Api\UserApi'
echo $stub->stubClass;       // 'MyApp\Api\UserApiStub'
echo $stub->stubShortName;   // 'UserApiStub'
echo $stub->stubNamespace;   // 'MyApp\Api'
echo $stub->filePath;        // '/var/www/src/Generated/Api/UserApiStub.php'
echo $stub;                  // 'MyApp\Api\UserApiStub' (via __toString)
```

### `generateAll(array $interfaceClasses): GeneratedStub[]`

Generates multiple stubs in one call.

```php
$stubs = $generator->generateAll([
    UserApi::class,
    ProductApi::class,
    OrderApi::class,
]);

foreach ($stubs as $stub) {
    echo "{$stub->interfaceClass} → {$stub->filePath}\n";
}
```

### What Gets Generated

The generated class is:

- `final` — not meant to be extended
- Has a single `__construct(SdkClient $client)` constructor
- Implements the source interface
- Has one method for each annotated interface method
- Each method builds a `CallDescriptor` and delegates to `$this->client->execute()`

Example output for a `#[GET('/users/{id}')]` method:

```php
// AUTO-GENERATED by Tugrul\ApiGen\Generator\StubGenerator. Do not edit manually.

namespace MyApp\Api;

use Tugrul\ApiGen\Contracts\SdkClient;
use Tugrul\ApiGen\Http\CallDescriptor;
use MyApp\Api\UserApi;

final class UserApiStub implements UserApi
{
    public function __construct(private readonly SdkClient $client) {}

    public function getUser(int $id): array
    {
        $call = CallDescriptor::create()
            ->method('GET')
            ->path('/users/{id}')
            ->pathParam('id', $id)
            ->returnType('array');

        return $this->client->execute($call);
    }

    public function deleteUser(int $id): void
    {
        $call = CallDescriptor::create()
            ->method('DELETE')
            ->path('/users/{id}')
            ->pathParam('id', $id);

        $this->client->execute($call);   // no 'return' — void method
    }
}
```

> **Note on `void` methods:** When the interface method has a `void` return type, the generated code calls `$this->client->execute()` without `return`. This keeps static analysis tools (PHPStan, Psalm) happy.

## Naming Strategies

A naming strategy determines the **class name** and **namespace** of the generated stub. Pass any `StubNamingStrategy` instance to `StubGenerator`.

### `DefaultNamingStrategy`

Mirrors the interface exactly — same namespace, same class name. If a file already exists at the resolved output path and it is **not** a previously generated stub (detected by the `AUTO-GENERATED` marker), appends `Impl` to avoid overwriting hand-written code.

```php
// MyApp\Api\UserApi  →  MyApp\Api\UserApi
// (MyApp\Api\UserApiImpl if UserApi.php already exists and is hand-written)
new DefaultNamingStrategy()
```

### `SuffixNamingStrategy`

Appends a fixed suffix to the class name. Optionally overrides the namespace.

```php
new SuffixNamingStrategy('Stub')
// MyApp\Api\UserApi  →  MyApp\Api\UserApiStub

new SuffixNamingStrategy('Client')
// MyApp\Api\UserApi  →  MyApp\Api\UserApiClient

new SuffixNamingStrategy('Stub', 'MyApp\Clients')
// MyApp\Api\UserApi  →  MyApp\Clients\UserApiStub
```

### `SubNamespaceNamingStrategy`

Inserts a child namespace segment after the interface's namespace. Optionally appends a suffix.

```php
new SubNamespaceNamingStrategy('Generated')
// MyApp\Api\UserApi  →  MyApp\Api\Generated\UserApi

new SubNamespaceNamingStrategy('Generated', 'Stub')
// MyApp\Api\UserApi  →  MyApp\Api\Generated\UserApiStub
```

### `CallableNamingStrategy`

Full control via a closure. The closure receives the `ReflectionClass` of the interface and the output directory string, and must return `[namespace, shortName]`.

```php
new CallableNamingStrategy(function (\ReflectionClass $iface, string $outputDir): array {
    $date = date('Ymd');
    return [
        $iface->getNamespaceName() . '\\Generated',
        $date . '_' . $iface->getShortName(),
    ];
});
// MyApp\Api\UserApi  →  MyApp\Api\Generated\20240301_UserApi
```

### `MappedNamingStrategy`

Resolves naming from a static map keyed by interface FQCN. Unmapped interfaces are handled by a fallback strategy (default: `DefaultNamingStrategy`).

```php
new MappedNamingStrategy(
    map: [
        UserApi::class    => ['MyApp\Clients', 'UserClient'],
        ProductApi::class => ['MyApp\Clients', 'ProductClient'],
    ],
    fallback: new SuffixNamingStrategy('Stub'),
);
// UserApi    →  MyApp\Clients\UserClient
// ProductApi →  MyApp\Clients\ProductClient
// OrderApi   →  MyApp\Api\OrderApiStub   (fallback)
```

### Summary Table

| Strategy | Constructor | Result for `MyApp\Api\UserApi` |
|---|---|---|
| `DefaultNamingStrategy` | `()` | `MyApp\Api\UserApi` |
| `SuffixNamingStrategy` | `('Stub')` | `MyApp\Api\UserApiStub` |
| `SubNamespaceNamingStrategy` | `('Generated')` | `MyApp\Api\Generated\UserApi` |
| `SubNamespaceNamingStrategy` | `('Generated', 'Stub')` | `MyApp\Api\Generated\UserApiStub` |
| `CallableNamingStrategy` | `(fn(...) => ...)` | Whatever the closure returns |
| `MappedNamingStrategy` | `([...], fallback: ...)` | Per-interface explicit mapping |

## PSR-4 Aware Output Paths

### The Problem

PSR-4 maps a namespace prefix to a root directory:

```json
"autoload": { "psr-4": { "App\\": "src/" } }
```

This means `App\Api\UserApi` lives at `src/Api/UserApi.php`. A naive namespace-to-path conversion would produce:

```
outputDir/App/Api/UserApiStub.php   ← WRONG: double-nests "App"
```

The correct path is:

```
outputDir/Api/UserApiStub.php       ← CORRECT: strips the PSR-4 root prefix
```

### How OutputPathResolver Works

`OutputPathResolver` detects the PSR-4 root prefix and strips it from the stub namespace before computing the output path. It uses two strategies in priority order:

**1. Composer PSR-4 map** *(preferred)* — reads `autoload.psr-4` and `autoload-dev.psr-4` from `composer.json`. The longest matching prefix wins.

**2. Reflection heuristic** *(fallback)* — compares namespace segments against actual directory segments from the right side until they diverge. The divergence point is the PSR-4 root boundary.

```
FQCN:    Vendor\Pkg\Api\UserApi
File:    /project/src/Api/UserApi.php
Dirs:    [project, src, Api]
NS segs: [Vendor, Pkg, Api]

Walk right-to-left:
  "Api" == "Api"  ✓ match
  "Pkg" == "src"  ✗ diverge  →  root prefix = "Vendor\Pkg"

Strip root:  "Api\UserApi"  →  output: outputDir/Api/UserApiStub.php
```

### Auto-discovery (default)

By default, `StubGenerator` walks up the directory tree from the output directory to find `composer.json` automatically. No configuration needed in most projects:

```php
// Auto-discovery — just omit the third argument
$generator = new StubGenerator('/var/www/src/Generated', new SuffixNamingStrategy('Stub'));
```

### Explicit Configuration

```php
use Tugrul\ApiGen\Generator\OutputPathResolver;

// Explicit composer.json path
$resolver = new OutputPathResolver(
    outputDir:        '/var/www/src/Generated',
    fallbackMode:     OutputPathResolver::FALLBACK_FLAT,
    composerJsonPath: __DIR__ . '/composer.json',
);

$generator = new StubGenerator('/var/www/src/Generated', $naming, $resolver);
```

### Fallback Modes

When the PSR-4 root cannot be determined (no `composer.json` found, built-in class, PHAR):

| Mode | Constant | Behaviour |
|---|---|---|
| Flat *(default)* | `FALLBACK_FLAT` | File goes directly in `outputDir`. |
| Full namespace | `FALLBACK_FULL_NAMESPACE` | Full namespace used as sub-path (old naive behaviour). |

```php
OutputPathResolver::FALLBACK_FLAT             // outputDir/UserApiStub.php
OutputPathResolver::FALLBACK_FULL_NAMESPACE   // outputDir/MyApp/Api/UserApiStub.php
```

### Static Helpers

```php
// Find composer.json by walking up from a directory
$path = OutputPathResolver::findComposerJson(__DIR__);  // ?string

// Auto-discover and build a resolver in one call
$resolver = OutputPathResolver::withAutoDiscoveredComposer(
    outputDir:    '/var/www/generated',
    startDir:     __DIR__,
    fallbackMode: OutputPathResolver::FALLBACK_FLAT,
);
```

## Conflict Detection

If `DefaultNamingStrategy` is used and a file already exists at the resolved output path:

- If the file contains the `AUTO-GENERATED by Tugrul\ApiGen` marker → **overwrite** (it's a previously generated stub).
- If the file does **not** contain the marker → **append `Impl` suffix** (protect hand-written code).

This detection is performed in `StubGenerator::generate()` after the output path is computed, since only the generator has access to both the naming result and the resolved path.
